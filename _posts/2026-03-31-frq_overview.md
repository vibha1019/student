# Sprint 9 – FRQ Learning Reflection (Individual Summary)

## Overview

Throughout this course, I completed multiple AP Computer Science A Free Response Question (FRQ) assignments designed to strengthen my understanding of Java programming concepts, including object-oriented design, data structures, and algorithmic problem solving. These FRQs progressively increased in complexity and helped me develop the ability to translate written problem statements into structured, efficient code. Over time, I improved both my accuracy and my ability to recognize patterns in common FRQ structures.

---

## Key FRQ Topics Covered

### ArrayList and List Processing
A major focus of FRQ practice involved working with `ArrayList` and list-based data manipulation. These problems required adding, removing, and iterating through elements while maintaining correct logic and structure. I practiced filtering data, searching for specific values, and modifying lists safely during iteration.

These types of problems strengthened my understanding of:
- Iteration patterns using loops
- Safe modification of lists
- Linear search techniques
- Conditional filtering logic

---

### String Manipulation and Parsing
Another major topic covered was string processing, especially problems involving delimiters and structured text. These FRQs required splitting strings, extracting meaningful parts, and handling edge cases such as empty inputs or unexpected formats.

Through these exercises, I improved my ability to:
- Parse structured strings
- Use substring and character-level logic
- Handle edge cases carefully
- Debug off-by-one and formatting errors

---

### Object-Oriented Programming (OOP) FRQs
Several FRQ-style problems focused on designing and implementing classes. These required defining fields, constructors, and methods that model real-world systems.

From these problems, I strengthened my understanding of:
- Encapsulation through private fields and public methods
- Class design and structure
- Method implementation and return values
- Using objects to represent data and behavior

---

### Algorithmic Thinking and Problem Solving
Across all FRQs, a consistent skill developed was algorithmic thinking. Many problems required step-by-step logic, nested loops, or simulation-style reasoning.

This helped me improve:
- Breaking down complex problems into smaller steps
- Writing structured pseudocode before coding
- Debugging logical errors systematically
- Recognizing common patterns in FRQ design

---

## Skills Developed

Through repeated FRQ practice, I improved significantly in the following areas:
- Translating problem descriptions into Java code
- Selecting appropriate data structures such as ArrayLists and classes
- Writing clean and structured iterative logic
- Debugging edge cases and logical errors
- Managing time and efficiency under FRQ-style constraints

---

## Challenges and Growth

One of the main challenges I faced was handling edge cases correctly, especially in list iteration and string parsing problems. Early on, I made mistakes such as off-by-one errors or failing to account for empty inputs.

Over time, I improved by:
- Testing edge cases more systematically
- Tracing code execution step-by-step
- Breaking problems into smaller logical parts
- Reviewing patterns from previous FRQs to avoid repeated mistakes

---

## Connection to Sprint 9 and CS 113 Requirements

These FRQ practices directly support Sprint 9 and CS 113 learning objectives:

- **Data Structures:** Use of ArrayLists for dynamic data storage and manipulation  
- **Algorithms:** Implementation of searching, iteration, and structured logic  
- **Object-Oriented Design:** Class creation, encapsulation, and method design  
- **Debugging and Testing:** Iterative refinement and validation of solutions  

---

## Final Reflection

Overall, FRQ practice significantly strengthened my understanding of Java and improved my ability to think algorithmically. These skills are directly applicable to my capstone project, especially in backend logic design, data structure usage, and problem decomposition. This experience has also better prepared me for both the AP CSA exam and future software development work.
